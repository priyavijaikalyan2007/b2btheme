6. Auto-discovered library components ADR-134

            

                Field type values like "cron-picker",
                "color-picker", "date-picker",
                "angle-picker" are auto-resolved at render time to
                window.createCronPicker etc. — zero adapter code.
                Real CDN components, real values flowing through
                getValues().
            

            

            

                
                

            

        


        

            
7. Registry override — non-conformant component ADR-134

            

                Demonstrates registerDynamicFormFieldProvider():
                we register a synthetic "slider-with-label" type
                whose factory is built inline from native HTML. The variant
                uses it like any built-in.
            

            

            

                
                

            

        


        

            
8. Composite mount field — editor + help pane ADR-134

            

                One field whose mount(host) builds a custom split
                layout containing a CodeEditor on the left and a help pane on
                the right. DynamicFormSwitcher drives lifecycle through the
                returned adapter.
            

            

            

                
                

            

        


        

            
9. Literate error — missing factory ADR-134

            

                A variant declaring type: "nosuch-picker" for which
                no factory is loaded and no provider is registered. The
                component renders an in-place literate error naming the
                expected factory and the registry escape hatch.
