# ChartPanel — per-component progress log

Resume-friendly session log for the ChartPanel component (slug `chartpanel`). For the API spec see `components/chartpanel/README.md`; for the design rationale see ADR-130 in `agentknowledge/decisions.yaml`.

## Status (as of 2026-04-27)

- **Shipped:** 2026-04-27 (ADR-130).
- **Tests:** 23 component tests with a Chart.js mock, all green.
- **Build:** `npm run build` clean. Dist artifacts at `dist/components/chartpanel/{chartpanel.css, .js, .d.ts}`. Built JS ~9 KB raw / **3.1 KB gzip** (target was ≤8 KB gzip; comfortable headroom).
- **Adopted by:** none yet. First adopter target: Tenant Admin analytics pages (`frontend/admin/analytics-*.html`). Subsequent adopters: Strukture analytics, Diagrams metrics overlay.
- **External dependency:** `window.Chart` (Chart.js >= 4.4, < 5). Vendored at `dist/vendor/chart.js/chart.umd.min.js` for our own demos via `npm run copy:js`.
- **Stencil:** Tier C entry in `stencils-ui-components.ts` with custom render fn (axis frame + bars + gridlines).
- **Component Studio:** entry in the *Feedback* category.
- **Demo:** `demo/components/chartpanel.html` (standalone) + section in `demo/all-components.html`.

## Origin

Filed by the apps team on 2026-04-27 in `specs/2026-04-27-tenant-admin-cdn-components.md` together with MetricCard. Initial spec asked for Chart.js to be bundled inside the wrapper; UI-team review re-routed to the established external-globals pattern.

## Decisions captured

| Decision | Outcome |
|---|---|
| Chart.js loading | **ADR-028 external-globals pattern** — wrapper reads `window.Chart`, host loads Chart.js separately. Same as CodeMirror via `codeeditor.ts`, marked / hljs / katex / mermaid / Viz via `markdownrenderer.ts`. Bundling inside would have required adding esbuild / rollup to the build pipeline (currently just tsc + wrap-iife.sh + sass) for one component. |
| Slug | Renamed from spec's `Chart` to **`chartpanel`** to avoid shadowing the global `Chart` constructor that Chart.js installs on `window`. Pattern matches `helpdrawer` / `commandpalette` / `formdialog` (descriptive of role, not dependency). |
| Theme awareness | Resolve `--theme-*` tokens via `getComputedStyle(documentElement)` at every render. `MutationObserver` on `<html>[data-bs-theme]` (`themetoggle.ts:219` is the canonical writer) re-renders without animation on theme toggle. |
| `setData` semantics | Same series IDs → in-place `chart.update("none")` (no animation, single frame, no jank on poll-driven dashboards). Different IDs → rebuild Chart.js instance. |
| Required `ariaLabel` | TS-enforced. Chart is not decorative. Visually-hidden `<table>` fallback default-on for screen readers. |
| `exportSVG` | Dropped. Chart.js renders to canvas; SVG is non-trivial. Server-side path is the option for callers who need SVG. |
| Hex fallback constants | Kept (`FALLBACK_INTENT_HEX` in `chartpanel.ts`). Only fire when `getComputedStyle` returns empty (jsdom, very early page load). Runtime paint always tries `--theme-*` first. ADR-130 amended with this rationale so the next reader doesn't flag it as drift. |
| Demo Chart.js source | Switched from CDN (`cdn.jsdelivr.net`) to vendored (`dist/vendor/chart.js/chart.umd.min.js`) after the user reported the literate error in environments where the CDN was blocked or offline. Mirrors the existing Bootstrap vendoring (`copy:js` step). |
| Synthetic missing-dep demo card | Removed. Logged a `[ERROR]` console line indistinguishable from a real failure, and left a tempting `delete window.Chart` snippet on a public demo. Literate error path is documented in the README. |

## Tooltip on sparkline

Chart.js tooltips are enabled on sparkline as well (`tooltip.enabled: true`). Originally written as `!isSpark || true` which always evaluates `true` — caught in advisor review and simplified.

## Open / parked

- Pie / donut / radar charts — out of scope for v1 per spec.
- Stacked / grouped bar variants — out of scope for v1.
- Drill-down / linked charts — Tenant Admin doesn't need them for v1.
- Server-side SVG export path — out of scope; document if/when an adopter needs it.
